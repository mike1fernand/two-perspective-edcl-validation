# Data availability and artifacts

This repository is structured so that heavy generated outputs do not live in normal git history.

The key distinction is:

```text
git:
  source code, templates, small deterministic artifacts, compact result summaries

GitHub Release assets:
  heavy chains, timestamped workdirs, bundles, patched runtime/build outputs if needed
```

## What is in git

The repository should contain:

```text
code
configs
Cobaya YAML templates
CLASS patch files
small deterministic Tier-B / Track-0 artifacts
small Tier-A1 result summaries
documentation
```

Important compact Tier-A1 result files in git:

```text
cosmology/results/tierA1_hubble_result_card.json
cosmology/results/tierA1_chain_component_audit.json
```

Important Tier-A1 reproducibility files in git:

```text
cosmology/patches/class_edcl.patch
cosmology/cobaya/edcl_cosmo_lateonly.yaml.in
cosmology/cobaya/edcl_cosmo_full.yaml.in
cosmology/scripts/run_tiera1_lateonly_suite.py
cosmology/scripts/check_no_doublecount_sh0es.py
cosmology/scripts/validate_tiera1_lateonly_results.py
cosmology/likelihoods/H0_edcl_func.py
```

## What should not be committed

Do not commit generated heavy/runtime outputs:

```text
class_public/
cobaya_packages/
chains/
edcl_tiera1_*/
bundle_edcl_tiera1.zip
*.updated.yaml
__pycache__/
*.pyc
```

## What should be published as GitHub Release assets

Publish heavy Tier-A1 reproducibility artifacts as Release assets when they are needed for external verification:

```text
Tier-A1 chain outputs
Tier-A1 timestamped workdirs
Tier-A1 bundle zips produced by the runner
checksums
release-asset README
```

The corrected Tier-A1 runner writes a timestamped workdir such as:

```text
edcl_tiera1_YYYYMMDD_HHMMSS/
```

Expected corrected workdir contents:

```text
manifest.json
logs/
yamls/
chains/
results_summary.json
results_report.md
bundle_edcl_tiera1.zip
```

The bundle generated by the runner is expected at:

```text
<workdir>/bundle_edcl_tiera1.zip
```

## Minimum useful Tier-A1 release asset

A practical release asset can be one zip, for example:

```text
tierA1_reproducibility_assets.zip
```

Suggested contents:

```text
chains/lcdm_production.1.txt
chains/edcl_production.1.txt
chains/edcl_no_h0_medium.1.txt
workdirs/<corrected-run-workdir>/
checksums/SHA256SUMS.txt
README_RELEASE_ASSETS.md
```

If older timestamped workdirs are unavailable, publish the available chains plus a regenerated corrected-run workdir bundle and state clearly that original workdir-backed provenance remains unavailable.

## Checksums

Release assets should include SHA256 checksums for chain files and bundle files.

Use a checksums file such as:

```text
checksums/SHA256SUMS.txt
```

or the existing repo release-assets checksum convention if present.
